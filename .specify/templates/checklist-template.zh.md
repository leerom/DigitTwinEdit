## Constitution 检查

本检查清单应包含以下 Constitution 合规性检查项:

### 中文优先检查
- [ ] 所有文档使用中文编写
- [ ] 代码注释使用中文
- [ ] 用户界面文本使用中文
- [ ] 错误信息和日志使用中文

### 测试驱动开发检查
- [ ] 测试先于实现编写
- [ ] 测试失败后再实现功能
- [ ] 测试覆盖率 ≥ 90%
- [ ] CI/CD 测试门禁已配置

### 性能检查
- [ ] UI 响应时间 ≤ 100ms
- [ ] 渲染帧率 ≥ 60fps (如适用)
- [ ] 性能监控已实施
- [ ] 无主线程阻塞操作

### 架构检查
- [ ] 使用了适当的设计模式
- [ ] 组件间依赖注入
- [ ] 避免单例模式
- [ ] 代码可测试性良好

### 简单性检查
- [ ] 每个函数/类单一职责
- [ ] 避免过早抽象
- [ ] 代码自解释清晰
- [ ] MVP 范围明确
